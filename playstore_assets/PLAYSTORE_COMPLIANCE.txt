[YOU MUST DO BEFORE SUBMITTING]

[ ] Step 1: Generate your signing keystore
    Run: cd playstore_assets && ./generate_keystore.sh
    Save the .jks file in a safe location - NEVER lose it.

[ ] Step 2: Enable signing in app/build.gradle
    Uncomment the signingConfigs.release block and fill in:
    - storeFile path
    - storePassword
    - keyAlias
    - keyPassword
    Then uncomment: signingConfig signingConfigs.release

[ ] Step 3: Build a release AAB (not APK)
    In Android Studio:
    Build -> Generate Signed Bundle/APK -> Android App Bundle -> release

[ ] Step 4: Host the privacy policy
    Upload playstore_assets/privacy_policy.html to GitHub Pages:
    1. Create repo: github.com/new -> name: autexel-privacy
    2. Upload file as index.html
    3. Settings -> Pages -> Deploy from main branch
    4. Your URL: https://YOUR-USERNAME.github.io/autexel-privacy

[ ] Step 5: Take screenshots (minimum 2, recommended 4-6)
    Take these on a real device:
    a) Splash screen
    b) Home screen (both buttons visible)
    c) Camera scanning screen
    d) Excel table editor with data
    e) Invoice editor with items filled in
    f) Success state (file saved, Open File button visible)
    Screenshot size: at least 320px on shortest side, max 3840px

[ ] Step 6: Create Play Console account
    URL: play.google.com/console
    Fee: one-time USD $25
    Verification: takes 1-3 business days

[ ] Step 7: Create the app in Play Console
    - App name: Autexel - Scan to Excel and Invoice
    - Category: Productivity
    - Free app
    - Content rating: complete the questionnaire (select "Everyone")
    - Target audience: 18+ (simplest path, avoid extra child policy review)

[ ] Step 8: Fill the store listing
    Use the text in store_listing.txt - copy each section exactly.
    Upload:
    - High-res icon: playstore_icon_512.png
    - Feature graphic: playstore_feature_graphic_1024x500.png
    - Screenshots from Step 5

[ ] Step 9: Declare app content
    In Play Console -> App content:
    - Privacy policy: paste your hosted URL from Step 4
    - Ads: No
    - App access: All functionality is accessible
    - Content rating: Complete questionnaire -> Violence: No, Sex: No, etc.
    - Target audience: Adults (18+)
    - News app: No
    - COVID-19 contact tracing: No
    - Data safety: Fill accurately (see DATA SAFETY section below)

[ ] Step 10: Upload AAB and submit
    Release -> Production -> Create new release
    Upload your .aab file
    Add release notes (copy from store_listing.txt)
    Review -> Start rollout to Production

COMMON REJECTION REASONS AND HOW THIS APP AVOIDS THEM

1. SPAM / LOW QUALITY
   Avoided: The app has a clear, specific purpose and polished UI.

2. MISLEADING METADATA
   Avoided: Store listing accurately describes what the app does.
   Do not add keywords unrelated to the app's function.

3. PRIVACY POLICY MISSING OR INADEQUATE
   Avoided: Full privacy policy is included. Host it before submitting.
   It must be accessible via a live URL - not a file attachment.

4. PERMISSIONS NOT JUSTIFIED
   Avoided: Camera and Storage are both essential and declared.
   No sensitive permissions are requested unnecessarily.

5. TARGET SDK TOO LOW
   Avoided: targetSdk = 34, above Google's minimum requirement.

6. SIGNING ISSUES
   Avoided: Use release signing config. Debug-signed AABs are rejected.

7. CRASHES ON LAUNCH
   Test on a real device before submitting.
   Test: launch, grant camera permission, scan, export both formats.

8. CONTENT RATING MISMATCH
   Avoided: App contains no violence, sexual content, gambling, or hate.
   Rate it as "Everyone" in the content questionnaire.
